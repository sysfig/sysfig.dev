config.toml

// 14 Dec 20
This post is part B1 of the series Generate Static Sites with Hugo.

To start off, we have the config.toml configuration file.

baseURL = "http://example.org/"
languageCode = "en-us"
title = "My New Hugo Site"

config.toml holds variables that can be used in all templates. Typically, these would be site-specific information like the site’s name, canonical domain name, language code, theme, etc.

By default, the configuration is written in Tom’s Obvious, Minimal Language (TOML), named after Tom Preston-Werner, the co-founder of GitHub and Jekyll. But if you’re like me and prefer to write configuration files in YAML Ain’t Markup Language (YAML), then you can delete the whole directory and re-create the site using the --format option in the hugo new site command.

hugo new site sysfig.dev --format yaml

The rest of this book will use YAML for the configuration file. Now let’s look at what these three configuration variables mean:

  • baseURL - used to construct absolute URLs. This should be the website’s canonical domain name.
  • languageCode - For accessibility to allow screen readers to pronounce the contents of the page as well as the page’s <title> element appropriately. It should be a valid IETF identifying language tag

You can find a list of all valid configuration settings on the Configure Hugo page.
